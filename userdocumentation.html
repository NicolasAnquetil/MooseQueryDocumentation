Introduction

Moose-Query is a domain-specific language to build queries for entities in Moose. It succeed to Moose-Chef. It is designed to simplified and standardize inside a Moose model.

An application model in Moose is composed of two concepts:

Imbrication of entities
Define which entity contains which entities. For example it will define the classes contained in a package and methods contained in a class. In the example image, it can be the relations "parentPackage", "container" or "parentType"

Interactions between entities
Define how the entities interact with each other. For example a inheritance is an interaction between a subclass and its superclass. Another example is that a reference is an interaction between a behavioural entity and a type. In the example image it can be the :Inheritance with the "superclass" and "subclass" relations.

MooseQuery allow to explore a model via those two concepts. With it you will be able to:

  • The exploration of the containment tree
  • The gathering of associations
  • The manipulation of those gathered associations
  • The change of scope

In this documentation we will explore the use of the DSL. At the end you will be able to query your moose model, for example, to:

  • Get the children of an entity (Exploration of the containment tree)
  • Get all the references contained in a class (Gathering of associations)
  • Get all the entities who depend on a specific entity (Manipulation of gathered associations)
  • Get all the methods contained in a package in a Java model, including those in inner/anonymous classes (Change of scope)

Containment tree exploration

Exploring the containment tree is the easiest way to query a model. It will be possible with the imbrication relations in the model. To do so, it exist a really simple API on entities:

#children    Will return the direct children of the entity
#parents    Will return the direct parents of the entity
#allChildren    Will return the children of the entity and there children recursively
#allParents    Will return the parents of the entity and there parents recursively

For example, if we take the model present in the figure as example we can do:

Containment tree examples

package1 children. "=> { package2 . class1 }"
class3 children. "=> { attribut1 . attribut2 }"

package1 allChildren. "=> { package2 . class1 . class2 . class3 . attribut1 . attribut2 }"
class3 allChildren. "=> { attribut1 . attribut2 }"

package1 parents. "=> { }"
class3 parents. "=> { package2 }"

class3 allParents. "=> { package2 . package1 }"
attribut1 allParents. "=> { class3 . package2 . package1 }"
